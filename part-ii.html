개발자 API

Apple Accessibility 55가지 기능을 개발자 관점에서 정리. UIKit UIAccessibility · SwiftUI 모디파이어 · 환경값부터 피쳐 전용 프레임워크까지, Apple의 5개 카테고리 구조로 배열한다.

01 UIKit · UIAccessibility

모든 UIKit 접근성의 시작점. UIView·UIBarItem이 자동으로 채택하며, 속성을 채우는 것만으로 VoiceOver · Voice Control · Switch Control이 연동된다.

핵심 속성 (Accessibility Attributes)

속성설명
isAccessibilityElement단일 접근성 요소로 노출 여부
accessibilityLabel요소의 짧은 이름
accessibilityAttributedLabel발음·언어 힌트 포함 속성형 레이블
accessibilityValue슬라이더 값, 텍스트필드 내용 등 상태값
accessibilityHint"두 번 탭하면 재생됩니다" 같은 행동 결과 힌트
accessibilityTraits.button .header .selected .adjustable .link .playsSound .startsMediaSession 등 비트마스크
accessibilityIdentifierUI 테스트용 ID (VoiceOver 출력에는 미사용)
accessibilityFrame / accessibilityActivationPoint화면상 영역과 두 번 탭 활성화 지점
accessibilityElementsHidden하위 요소 전체 숨김
accessibilityViewIsModal모달 표시 (형제 요소 차단)
accessibilityLanguage"ko-KR" 같은 BCP-47 코드로 발음 언어 지정
accessibilityCustomActions로터에 추가되는 커스텀 액션 배열
accessibilityCustomRotors커스텀 로터 (예: "헤딩", "이미지")
accessibilityCustomContentiOS 14+ More Content 로터 항목
shouldGroupAccessibilityChildren자식들을 한 그룹으로 묶어 순서 보장
accessibilityUserInputLabelsVoice Control 명령에 사용할 후보 이름 배열
accessibilityContainerType.list .dataTable .semanticGroup .landmark
accessibilityShowsLargeContentViewer탭 앤 홀드 Large Content Viewer 활성화
accessibilityTextualContext.console .messaging .sourceCode .narrative 등 발음 전략 힌트
accessibilityDirectTouchOptions피아노·드로잉처럼 직접 터치가 의미를 갖는 영역
 
let heartButton = UIButton()
heartButton.accessibilityLabel = "좋아요"
heartButton.accessibilityValue = isLiked ? "눌림" : "안 눌림"
heartButton.accessibilityHint = "두 번 탭하면 게시물을 좋아합니다"
heartButton.accessibilityTraits = [.button]
heartButton.accessibilityUserInputLabels = ["좋아요", "하트", "Like"]

컨테이너 프로토콜

  • UIAccessibilityContainer — 자식 요소 직접 관리 (accessibilityElements, accessibilityElementCount())
  • UIAccessibilityContainerDataTable / ...DataTableCell — 행·열 정보 있는 표
  • UIAccessibilityReadingContent — 페이지 가진 텍스트 (스크롤 가능 책 뷰 등)
  • UIAccessibilityFocus — 포커스 진입·이탈 시점 콜백
  • UIAccessibilityAction — magic tap, escape, increment, decrement 등 시스템 액션
override var accessibilityTraits: UIAccessibilityTraits {
    get { [.adjustable] }
    set {}
}
override func accessibilityIncrement() { value += step; updateUI() }
override func accessibilityDecrement() { value -= step; updateUI() }

02 SwiftUI 접근성 Modifier 카탈로그

표준 컴포넌트는 자동으로 접근성 레이블을 만든다. 우리가 개입하는 것은 (1) 자동 추론이 부족한 곳, (2) 커스텀 그리기/제스처 영역, (3) 의미 그룹화·액션 추가이다.

Image(systemName: "heart.fill")
    .accessibilityLabel("좋아요")
    .accessibilityValue(isLiked ? "눌림" : "안 눌림")
    .accessibilityHint("두 번 탭하면 좋아요")
    .accessibilityAddTraits(.isButton)
    .accessibilityInputLabels(["좋아요", "하트"])

HStack { Image(systemName: "sun.max"); Text("오늘 24°C 맑음") }
    .accessibilityElement(children: .combine)
    .accessibilityLabel("오늘 날씨, 맑음, 24도")

DecorativeIcon().accessibilityHidden(true)

ComplexChart()
    .accessibilityRepresentation { Text("매출 차트, 1분기 대비 12% 상승") }
Modifier용도
.accessibilityLabel대체 이름
.accessibilityValue현재 상태값
.accessibilityHint행동 결과 힌트
.accessibilityIdentifierUI 테스트 식별자
.accessibilityAddTraits / .accessibilityRemoveTraits.isButton .isHeader .isLink .isSelected .isModal .updatesFrequently .isToggle 등
.accessibilityHidden트리에서 제외
.accessibilityElement(children:).ignore / .combine / .contain
.accessibilityRepresentation { View }가상 뷰로 대체
.accessibilityAction(_:_:)커스텀 액션 + .magicTap .escape .default
.accessibilityAdjustableAction스와이프 업/다운 증감
.accessibilityRotor커스텀 로터 정의
.accessibilityCustomContentMore Content 로터 항목
.accessibilityHeading(.h1/.h2/...)헤딩 레벨
.accessibilityFocused($flag)프로그래매틱 포커스
.accessibilityShowsLargeContentViewer탭 앤 홀드 확대 뷰어
.accessibilityInputLabels(_:)Voice Control 별칭
.accessibilityIgnoresInvertColors색 반전 무시 (사진 등)
.accessibilityTextContentType(_:)console · sourceCode · spreadsheet 등
.speechSpellsOutCharacters()한 글자씩 발음
.speechAdjustedPitch(_:)피치 조정
.speechAlwaysIncludesPunctuation()구두점도 읽음
.speechAnnouncementsQueued()알림을 큐로 처리

03 SwiftUI Environment Values

사용자의 시스템 접근성 설정을 SwiftUI에서 읽기. 분기 처리와 애니메이션 비활성화에 모두 쓴다.

Environment Key의미
\.accessibilityVoiceOverEnabledVoiceOver 실행 여부
\.accessibilitySwitchControlEnabledSwitch Control 실행 여부
\.accessibilityReduceMotionMotion settings — 모션 줄이기
\.accessibilityPrefersCrossFadeTransitions슬라이드 대신 크로스 페이드
\.accessibilityReduceTransparency투명도 줄이기
\.accessibilityDifferentiateWithoutColor색만으로 구분하지 않기
\.accessibilityInvertColors색 반전
\.accessibilityShowButtonShapes버튼 모양 표시
\.accessibilityLargeContentViewerEnabledLarge Content Viewer 사용 가능
\.accessibilityPlayAnimatedImages애니메이션 이미지 재생 여부
\.legibilityWeightBold Text → .bold / .regular
\.colorSchemeContrastIncrease Contrast → .increased
\.dynamicTypeSize현재 Dynamic Type 크기 (.xSmall ~ .accessibility5)
\.sizeCategory (legacy)구버전 환경 키
 
@Environment(\.accessibilityReduceMotion) var reduceMotion
@Environment(\.dynamicTypeSize) var typeSize

Image("hero")
    .animation(reduceMotion ? nil : .spring, value: scale)

Text("제목").font(.title)
    .lineLimit(typeSize.isAccessibilitySize ? nil : 2)

Vision

시각 관련 16가지 기능의 개발자 API. VoiceOver가 중심이며, Dynamic Type · Zoom · Motion · Audio Descriptions 등이 각자의 API를 가진다.

VoiceOver

iOS 3+macOS 10.4+watchOS 7+visionOS 1+

화면 리더. 핵심 API는 Foundation 섹션의 UIAccessibility 속성과 SwiftUI Modifier이다. 여기서는 VoiceOver 전용 심화 기능을 다룬다.

커스텀 로터 (UIKit)

let rotor = UIAccessibilityCustomRotor(name: "중요 메시지") { predicate in
    let current = predicate.currentItem.targetElement as? MessageCell
    let next = findNext(after: current, direction: predicate.searchDirection)
    guard let cell = next else { return nil }
    return UIAccessibilityCustomRotorItemResult(targetElement: cell, targetRange: nil)
}
view.accessibilityCustomRotors = [rotor]

커스텀 로터 (SwiftUI)

List(messages) { msg in MessageRow(msg) }
    .accessibilityRotor("중요 메시지", entries: messages.filter(\.isImportant), entryID: \.id) { msg in
        Text(msg.preview)
    }

커스텀 액션

// UIKit
cell.accessibilityCustomActions = [
    UIAccessibilityCustomAction(name: "답장") { _ in reply(); return true },
    UIAccessibilityCustomAction(name: "삭제") { _ in delete(); return true }
]

// SwiftUI
MessageRow(message)
    .accessibilityAction(named: "답장") { reply() }
    .accessibilityAction(named: "삭제") { delete() }
    .accessibilityAction(.magicTap) { togglePlayback() }

포커스 제어 (SwiftUI)

@AccessibilityFocusState private var isErrorFocused: Bool

VStack {
    if showError {
        Text("전송 실패").accessibilityFocused($isErrorFocused)
    }
}.onChange(of: showError) { _, v in if v { isErrorFocused = true } }
Tip스와이프 액션이 있는 셀에는 반드시 같은 동작을 accessibilityCustomActions로도 제공해야 한다. 그렇지 않으면 VoiceOver 사용자는 그 기능에 접근할 수 없다.

Magnifier

iOS 10+

카메라로 피사체를 크게 보는 시스템 앱. 서드파티가 직접 호출할 수 있는 API는 없다. UIActivityViewController로 이미지를 공유하면 사용자가 Magnifier에서 열 수 있다.

개발자 할 일앱 내 이미지·텍스트는 @3x 고해상도를 유지한다. Magnifier가 확대했을 때 픽셀이 깨지지 않아야 한다.

Braille Access

iOS 10+macOS

MFi 점자 디스플레이 연동. 별도 SDK 없이 VoiceOver가 브리지 역할을 한다. 개발자는 accessibilityAttributedLabel에 UIAccessibilitySpeechAttributeIPANotation을 설정해 점자 변환 힌트를 줄 수 있다.

var attrs = AttributedString("Mgmt")
attrs[attrs.startIndex...].accessibilityTextCustom = "Management"   // 점자 전개
label.accessibilityAttributedLabel = NSAttributedString(attrs)

Accessibility Reader

iOS 18+iPadOS 18+

화면의 어느 텍스트든 읽어주는 시스템 읽기 모드. 별도 API 없이 앱의 시맨틱 구조(헤딩 트레잇, 논리적 읽기 순서)가 좋으면 더 잘 작동한다.

개발자 할 일.accessibilityHeading(.h1) 등 헤딩 트레잇을 올바르게 지정하고, accessibilityElements 순서가 읽기 흐름과 일치하도록 한다.

Read & Speak settings

iOS 8+

Speak Screen · Speak Selection · Typing Feedback 등 시스템 음성 읽기 설정. 앱 내 텍스트 발음을 세밀하게 제어할 때 SwiftUI Speech 모디파이어를 사용한다.

Modifier효과
.speechSpellsOutCharacters()한 글자씩 발음 (비밀번호 등)
.speechAlwaysIncludesPunctuation()구두점도 읽음
.speechAdjustedPitch(_ pitch: Double)피치 조정 (0.0–2.0)
.speechAnnouncementsQueued()알림을 큐로 처리 (겹침 방지)
 
// 상태 조회
UIAccessibility.isSpeakScreenEnabled
UIAccessibility.isSpeakSelectionEnabled

Display settings & Text Size

iOS 7+

Dynamic Type (7단계 일반 + 5단계 접근성), Bold Text, Increase Contrast, Smart Invert Colors, Grayscale. 앱이 이 설정들을 모두 존중해야 한다.

Dynamic Type — UIKit

label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true

// 커스텀 폰트 + 스케일
let base = UIFont(name: "Fraunces-Regular", size: 17)!
label.font = UIFontMetrics(forTextStyle: .body).scaledFont(for: base)

Dynamic Type — SwiftUI

Text("본문").font(.body)
Text("본문").font(.custom("Fraunces-Regular", size: 17, relativeTo: .body))

// 접근성 크기에서 레이아웃 전환
@Environment(\.dynamicTypeSize) var typeSize
if typeSize.isAccessibilitySize { VStack { ... } } else { HStack { ... } }

Bold Text / Increase Contrast / Invert

// UIKit
UIAccessibility.isBoldTextEnabled
UIAccessibility.isDarkerSystemColorsEnabled     // Increase Contrast
UIAccessibility.isInvertColorsEnabled
UIAccessibility.isGrayscaleEnabled

// SwiftUI Environment
@Environment(\.legibilityWeight) var weight      // .bold / .regular
@Environment(\.colorSchemeContrast) var contrast  // .increased / .standard
@Environment(\.accessibilityInvertColors) var invert

// 사진 등 색반전 제외
Image("photo").accessibilityIgnoresInvertColors(true)
Tip접근성 크기(.accessibilityMedium~.accessibility5)에서는 가로 레이아웃을 세로로 전환하는 게 정답일 때가 많다. ViewThatFits 또는 typeSize.isAccessibilitySize 분기로 대응한다.

Vehicle Motion Cues

iOS 17.4+

차량 이동 중 멀미를 줄이기 위해 화면 가장자리에 애니메이션 점을 표시하는 시스템 기능. 별도 개발자 API 없음. 앱이 accessibilityReduceMotion을 이미 존중하고 있다면 자동으로 조화를 이룬다.

Motion settings

iOS 7+

Reduce Motion, Cross-Fade Transitions. 모션 민감 사용자를 위해 앱의 애니메이션을 비활성화하거나 대체한다.

// UIKit
UIAccessibility.isReduceMotionEnabled
UIAccessibility.prefersCrossFadeTransitions

// SwiftUI
@Environment(\.accessibilityReduceMotion) var reduceMotion
@Environment(\.accessibilityPrefersCrossFadeTransitions) var crossFade

Image("hero")
    .scaleEffect(scale)
    .animation(reduceMotion ? nil : .spring(duration: 0.4), value: scale)
주의Lottie 등 서드파티 애니메이션 라이브러리는 Reduce Motion을 자동 감지하지 않는다. 직접 분기 처리해야 한다.

Zoom

iOS 5+macOS

화면 전체 또는 창 단위로 확대하는 시스템 줌. 개발자가 직접 제어할 API는 없으나, 탭 바나 커스텀 버튼처럼 작은 텍스트가 있는 곳에 Large Content Viewer를 구현하면 Zoom과 함께 사용하는 사용자 경험이 향상된다.

Large Content Viewer — UIKit

button.showsLargeContentViewer = true
button.largeContentTitle = "홈"
button.largeContentImage = UIImage(systemName: "house.fill")
button.addInteraction(UILargeContentViewerInteraction())

Large Content Viewer — SwiftUI

Label("홈", systemImage: "house.fill")
    .accessibilityShowsLargeContentViewer()

// 커스텀 콘텐츠
Image(systemName: "star")
    .accessibilityShowsLargeContentViewer {
        Label("즐겨찾기", systemImage: "star.fill")
    }
적용 대상시스템이 Large Content Viewer를 자동 처리하는 표준 컨트롤(UITabBar, UIToolbar 등)은 별도 구현 불필요. 커스텀 탭바·툴바에만 적용한다.

Hover settings

iPadOS 13.4+Mac CatalystmacOS

포인터(마우스·트랙패드·Apple Pencil Hover)에 반응하는 UI. Hover Typing은 iPad를 세워 놓고 트랙패드로 타이핑할 때 텍스트 커서 위 확대 뷰를 보여주는 시스템 기능으로, 별도 API 없음.

UIHoverGestureRecognizer (UIKit)

let hover = UIHoverGestureRecognizer(target: self, action: #selector(handleHover))
view.addGestureRecognizer(hover)

@objc func handleHover(_ g: UIHoverGestureRecognizer) {
    switch g.state {
    case .began, .changed: view.alpha = 0.85
    default: view.alpha = 1.0
    }
}

hoverEffect (SwiftUI)

Button("편집") { edit() }
    .hoverEffect(.lift)       // .automatic / .lift / .highlight

iWork

Pages · Numbers · Keynote 내 접근성 기능 (스크린 리더 지원, 대체 텍스트 등). Apple 내부 앱에 해당하므로 서드파티 개발자가 직접 호출할 API 없음.

Audio Descriptions

iOS 5+tvOS

영상 속 시각 정보를 음성으로 설명하는 오디오 트랙. AVKit이 자동으로 선택해 주므로, 미디어 파일에 오디오 설명 트랙이 포함되어 있으면 된다.

import AVFoundation

// 오디오 설명 트랙 존재 여부 확인
let asset = AVAsset(url: videoURL)
let group = try? await asset.loadMediaSelectionGroup(
    for: .audible
)
let adOptions = AVMediaSelectionGroup.mediaSelectionOptions(
    from: group?.options ?? [],
    with: [.AVMediaCharacteristicDescribesVideoForAccessibility]
)
// adOptions가 비어있지 않으면 오디오 설명 트랙 있음

Personal Voice

AVSpeechSynthesizer.requestPersonalVoiceAuthorization { status in
    guard status == .authorized else { return }

    let myVoice = AVSpeechSynthesisVoice.speechVoices()
        .first { $0.voiceTraits.contains(.isPersonalVoice) }

    guard let voice = myVoice else { return }

    var guidedAccessRestrictionIdentifiers: [String]? {
        ["com.myapp.restrict.chat"]
    }

        id == "com.myapp.restrict.chat" ? "채팅 비활성화" : nil

        chatEnabled = (state == .allow)
    }
}

Focus

iOS 15+

인지 부하를 줄이기 위한 집중 모드 지원. 앱은 SwiftUI @FocusState로 포커스를 프로그래매틱으로 이동시켜 불필요한 화면 요소를 줄인다. 시스템 집중 모드(방해 금지 등)와 연동은 FocusFilterIntent로 구현한다.

// SwiftUI 프로그래매틱 포커스
enum FormField { case name, email }
@FocusState private var focused: FormField?

Form {
    TextField("이름", text: $name).focused($focused, equals: .name)
    TextField("이메일", text: $email).focused($focused, equals: .email)
}.onAppear { focused = .name }

// 시스템 집중 모드 필터 (iOS 16+)
import AppIntents
struct MyFocusFilter: SetFocusFilterIntent {
    static var title: LocalizedStringResource = "MyApp 집중 필터"
    @Parameter(title: "알림 허용", default: true)
    var allowNotifications: Bool
    func perform() async throws -> some IntentResult { .result() }
}

07 Testing · Inspector · Audits

접근성은 "고쳤다고 믿었지만 안 고쳐진" 경우가 가장 흔한 분야. 도구로 자동 검증하고, 실제 VoiceOver로 직접 써 본다.

Accessibility Inspector

Xcode → Open Developer Tool → Accessibility Inspector. 시뮬레이터·실기기·Mac 앱 모두 검사 가능. Audit 탭에서 contrast, hit region, label, dynamic type, traits 누락 등을 자동 점검.

Xcode Accessibility Audit (XCTest)

iOS 17 / macOS 14부터 XCTest에 자동 감사가 내장됐다.

func testAccessibility() throws {
    let app = XCUIApplication()
    app.launch()
    try app.performAccessibilityAudit()           // 모든 카테고리
    try app.performAccessibilityAudit(
        for: [.contrast, .dynamicType, .hitRegion, .textClipping]
    )
}

Environment Overrides

Xcode → Environment Overrides에서 Bold Text · Increase Contrast · Reduce Motion · Reduce Transparency · Differentiate Without Color · Smart Invert · Larger Text를 라이브로 켜고 끌 수 있다.

VoiceOver 테스트 단축키

  • 실기기 VoiceOver 토글: Accessibility Shortcut(측면 버튼 3회) 또는 Settings → Accessibility → VoiceOver
  • macOS VoiceOver: ⌘ + F5
  • VoiceOver 수정키: VO = Control + Option
  • 항목 탐색: VO + →/←, 활성화: VO + Space
  • 로터: 두 손가락 회전 (실기기) / VO + U (macOS)

Vision Pro 접근성

응시 + 핀치가 기본 입력. 모든 인터랙티브 요소에 시각적 hover state와 충분한 hit target(권장 60pt 이상)이 필수. VoiceOver는 카메라 기반 공간 디스크립션과 Live Recognition을 함께 제공한다.
