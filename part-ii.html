개발자 API

우리 앱을 위 기능들과 연결하는 방법. UIKit의 UIAccessibility, SwiftUI의 모디파이어와 환경값, Dynamic Type, Speech 프레임워크, Music Haptics, Assistive Access API, 그리고 테스트 도구까지.

07 UIKit · UIAccessibility

UIKit에서 접근성은 UIAccessibility 프로토콜에 모여 있다. 거의 모든 UIView·UIBarItem이 채택하고 있어, 속성을 채우는 것만으로 VoiceOver가 읽어 준다.

핵심 속성 (Accessibility Attributes)

속성설명
isAccessibilityElement이 뷰가 단일 접근성 요소로 노출될지 여부
accessibilityLabel요소의 짧은 이름. "이미지 1" 같은 위치는 빼고 의미만
accessibilityAttributedLabel속성형 레이블(발음·언어 힌트)
accessibilityValue슬라이더 값, 텍스트필드 내용 등 상태값
accessibilityHint"두 번 탭하면 재생됩니다" 같은 행동 결과 힌트
accessibilityTraits.button, .header, .selected, .adjustable, .link, .playsSound, .startsMediaSession 등 비트마스크
accessibilityIdentifierUI 테스트용 ID(VoiceOver 출력엔 사용되지 않음)
accessibilityFrame / accessibilityActivationPoint요소의 화면상 영역과 두 번 탭 시 활성화 지점
accessibilityElementsHidden이 컨테이너의 하위 요소들을 모두 숨김
accessibilityViewIsModal이 뷰가 모달임을 표시(형제 요소들 차단)
accessibilityLanguage"ko-KR" 같은 BCP-47 코드로 발음 언어 지정
accessibilityCustomActions로터에 추가되는 커스텀 액션 배열
accessibilityCustomRotors커스텀 로터(예: "헤딩", "이미지")
accessibilityCustomContentiOS 14+ 커스텀 추가 정보. 사용자가 More Content 로터로 탐색
shouldGroupAccessibilityChildren자식들을 한 그룹으로 묶어 순서 보장
accessibilityRespondsToUserInteractionVoiceOver가 "탭 가능"한 요소로 인지하도록 강제
accessibilityUserInputLabelsVoice Control 명령에 사용할 후보 이름 배열
accessibilityContainerType.list, .dataTable, .semanticGroup, .landmark
accessibilityShowsLargeContentViewer탭 앤 홀드로 크게 보여주는 뷰어 활성화
accessibilityTextualContext.console, .fileSystem, .messaging, .narrative, .sourceCode, .spreadsheet, .wordProcessing — VoiceOver가 발음 전략을 조정
accessibilityDirectTouchOptions피아노·드로잉처럼 직접 터치가 의미를 갖는 영역

기본 예시

let heartButton = UIButton()
heartButton.setImage(UIImage(systemName: "heart.fill"), for: .normal)
heartButton.accessibilityLabel = "좋아요"
heartButton.accessibilityValue = isLiked ? "눌림" : "안 눌림"
heartButton.accessibilityHint = "두 번 탭하면 게시물을 좋아합니다"
heartButton.accessibilityTraits = [.button]
heartButton.accessibilityUserInputLabels = ["좋아요", "하트", "Like"]

컨테이너 프로토콜

  • UIAccessibilityContainer — 자식 요소를 직접 관리하는 컨테이너(accessibilityElements, accessibilityElementCount())
  • UIAccessibilityContainerDataTable / ...DataTableCell — 행·열 정보가 있는 표
  • UIAccessibilityContent — 페이지·문서 컨텐츠
  • UIAccessibilityReadingContent — 페이지를 가진 텍스트(스크롤 가능한 책 뷰 등)
  • UIAccessibilityFocus — 포커스 시점에 호출되는 콜백
  • UIAccessibilityAction — magic tap, escape, increment, decrement, scroll 같은 시스템 액션 처리
// Adjustable trait + accelerator/decrement
override var accessibilityTraits: UIAccessibilityTraits {
    get { [.adjustable] }
    set {}
}
override func accessibilityIncrement() { value += step; updateUI() }
override func accessibilityDecrement() { value -= step; updateUI() }

Large Content Viewer · Hover Style

탭 앤 홀드로 큰 텍스트/이미지 미리보기. showsLargeContentViewer = true와 largeContentTitle, largeContentImage를 설정하고 UILargeContentViewerInteraction을 추가한다.

08 SwiftUI 접근성 Modifier

SwiftUI는 거의 모든 표준 컴포넌트가 자동으로 접근성 레이블을 만든다. 우리가 손대는 것은 (1) 자동 추론이 부족한 곳, (2) 커스텀 그리기/제스처 영역, (3) 의미 그룹화·액션 추가이다.

레이블·값·힌트·트레잇

Image(systemName: "heart.fill")
    .accessibilityLabel("좋아요")
    .accessibilityValue(isLiked ? "눌림" : "안 눌림")
    .accessibilityHint("두 번 탭하면 좋아요")
    .accessibilityAddTraits(.isButton)
    .accessibilityIdentifier("like-button")
    .accessibilityInputLabels(["좋아요", "하트"])

그룹화·숨기기·표현 대체

HStack {
    Image(systemName: "sun.max")
    Text("오늘 24°C 맑음")
}
.accessibilityElement(children: .combine)   // 자식을 하나로 합침
.accessibilityLabel("오늘 날씨, 맑음, 24도")

DecorativeIcon().accessibilityHidden(true)

ComplexChart()
    .accessibilityRepresentation {          // 접근성용 가상 UI
        Text("매출 차트, 1분기 대비 12% 상승")
    }

커스텀 액션·로터

MessageRow(message)
    .accessibilityAction(named: "답장") { reply() }
    .accessibilityAction(named: "삭제") { delete() }
    .accessibilityAction(.magicTap) { togglePlayback() }
    .accessibilityAction(.escape) { dismiss() }

// Adjustable
Slider(value: $volume)
    .accessibilityAdjustableAction { direction in
        switch direction {
        case .increment: volume += 0.1
        case .decrement: volume -= 0.1
        @unknown default: break
        }
    }

커스텀 로터 (SwiftUI)

List(messages) { msg in MessageRow(msg) }
    .accessibilityRotor("중요 메시지", entries: messages.filter(\.isImportant), entryID: \.id) { msg in
        Text(msg.preview)
    }

포커스 제어

@AccessibilityFocusState private var isErrorFocused: Bool

VStack {
    if showError {
        Text("전송 실패")
            .accessibilityFocused($isErrorFocused)
    }
}
.onChange(of: showError) { _, newValue in
    if newValue { isErrorFocused = true }
}

09 SwiftUI Environment Values

사용자의 시스템 접근성 설정을 SwiftUI에서 읽기. 분기 처리와 애니메이션 비활성화에 모두 쓴다.

Environment Key의미
\.accessibilityEnabledVoiceOver/Switch Control 등 보조 기술 실행 중 여부
\.accessibilityVoiceOverEnabledVoiceOver 실행 여부
\.accessibilitySwitchControlEnabledSwitch Control 실행 여부
\.accessibilityReduceMotion모션 줄이기
\.accessibilityReduceTransparency투명도 줄이기
\.accessibilityDifferentiateWithoutColor색만으로 구분하지 않기
\.accessibilityInvertColors색 반전
\.accessibilityShowButtonShapes버튼 모양 표시
\.accessibilityLargeContentViewerEnabled큰 컨텐츠 뷰어 사용 가능
\.accessibilityPrefersCrossFadeTransitions슬라이드 대신 크로스 페이드
\.accessibilityPlayAnimatedImages애니메이션 이미지 재생 여부
\.legibilityWeightBold Text → .bold / .regular
\.colorSchemeContrastIncrease Contrast → .increased
\.dynamicTypeSize현재 Dynamic Type 크기(.xSmall ~ .accessibility5)
\.sizeCategory (legacy)구버전 환경 키
 
@Environment(\.accessibilityReduceMotion) var reduceMotion
@Environment(\.dynamicTypeSize) var typeSize

var body: some View {
    Image("hero")
        .scaleEffect(scale)
        .animation(reduceMotion ? nil : .spring, value: scale)

    Text("제목")
        .font(.title)
        .lineLimit(typeSize.isAccessibilitySize ? nil : 2)
}

10 Dynamic Type

텍스트 크기는 사용자가 시스템에서 정한다. 우리 앱은 그 선택을 존중하고, 7개의 일반 크기와 5개의 접근성 크기(accessibilityMedium~accessibility5)에서도 깨지지 않게 한다.

UIKit

label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true

// 커스텀 폰트 + 스케일
let base = UIFont(name: "Fraunces-Regular", size: 17)!
label.font = UIFontMetrics(forTextStyle: .body).scaledFont(for: base)

SwiftUI

Text("본문").font(.body)                  // 자동 스케일
Text("본문").font(.custom("Fraunces-Regular", size: 17, relativeTo: .body))

// 접근성 크기 제한 (가급적 사용 자제)
.dynamicTypeSize(.xSmall ... .accessibility2)
Tip 접근성 크기에서는 가로 레이아웃을 세로로 바꾸는 것이 정답일 때가 많다. ViewThatFits·typeSize.isAccessibilitySize 분기로 대응한다.

12 Custom Rotors & Custom Actions

VoiceOver 로터는 사용자가 두 손가락을 돌려 탐색 모드(헤딩·링크·이미지 등)를 바꾸는 회전 다이얼이다. 우리 앱의 의미 구조에 맞춰 커스텀 로터를 추가하면 탐색 효율이 크게 올라간다.

UIKit 커스텀 로터

let rotor = UIAccessibilityCustomRotor(name: "중요 메시지") { predicate in
    let current = predicate.currentItem.targetElement as? MessageCell
    let next = findNextImportantCell(after: current, direction: predicate.searchDirection)
    guard let cell = next else { return nil }
    return UIAccessibilityCustomRotorItemResult(targetElement: cell, targetRange: nil)
}
view.accessibilityCustomRotors = [rotor]

시스템 로터 타입 (UIAccessibility.CustomSystemRotorType)

.link, .visitedLink, .heading, .headingLevel1..6, .boldText, .italicText, .underlineText, .misspelledWord, .image, .textField, .table, .list, .landmark — 텍스트뷰에서 같은 의미의 영역을 자동 인덱싱.

Custom Actions

let reply = UIAccessibilityCustomAction(name: "답장") { _ in reply(); return true }
let delete = UIAccessibilityCustomAction(name: "삭제") { _ in delete(); return true }
cell.accessibilityCustomActions = [reply, delete]
Tip 스와이프 액션이 있는 셀에는 반드시 같은 동작을 accessibilityCustomActions로도 제공해야 한다. 그렇지 않으면 VoiceOver 사용자는 그 기능에 접근할 수 없다.

13 Speech · TTS · Personal Voice

텍스트 → 음성, 음성 → 텍스트, 그리고 사용자의 목소리로 합성하는 API.

AVSpeechSynthesizer (TTS)

import AVFoundation

let synth = AVSpeechSynthesizer()
let u = AVSpeechUtterance(string: "안녕하세요")
u.voice = AVSpeechSynthesisVoice(language: "ko-KR")
u.rate = AVSpeechUtteranceDefaultSpeechRate
u.pitchMultiplier = 1.0
u.preUtteranceDelay = 0.1
synth.speak(u)

Personal Voice

사용자가 디바이스에서 만든 Personal Voice를 받아오기. 사용자의 명시적 허가가 필요하다(AVSpeechSynthesizer.requestPersonalVoiceAuthorization).

AVSpeechSynthesizer.requestPersonalVoiceAuthorization { status in
    guard status == .authorized else { return }

    let personal = AVSpeechSynthesisVoice.speechVoices()
        .filter { $0.voiceTraits.contains(.isPersonalVoice) }

    if let myVoice = personal.first {
        let u = AVSpeechUtterance(string: "내 목소리로 말하기")
        u.voice = myVoice
        synth.speak(u)
    }
}
주의 Personal Voice는 Live Speech 같은 보조 통신 용도에서만 사용해야 한다. NSPersonalVoiceUsageDescription Info.plist 키와, 사용 목적을 명확히 밝힌 사용자 동의 흐름이 필수다.

SFSpeechRecognizer (STT)

import Speech

SFSpeechRecognizer.requestAuthorization { _ in }

let recognizer = SFSpeechRecognizer(locale: Locale(identifier: "ko-KR"))
let request = SFSpeechAudioBufferRecognitionRequest()
request.requiresOnDeviceRecognition = true      // 프라이버시·오프라인
recognizer?.recognitionTask(with: request) { result, _ in
    print(result?.bestTranscription.formattedString ?? "")
}

Listen for Atypical Speech

iOS 17.4+에서 시스템 음성 인식 모델이 비전형 발화에 적응 가능. 별도 API 호출은 필요 없고, 시스템 STT(SFSpeechRecognizer 포함)와 Siri에 자동 적용된다.

14 Music Haptics API

자체 음악 플레이어에서도 시스템 Music Haptics 트랙을 활용할 수 있게 해 주는 API. iOS 18+.

import MusicHaptics

Task {
    let manager = MusicHapticsManager.shared
    guard await manager.requestAuthorization() == .authorized else { return }

    if await manager.isHapticTrackAvailable(forSongID: "1234567890") {
        try await manager.play(songID: "1234567890", mode: .vocalsOnly, intensity: 0.8)
    }
}

모드는 .full·.vocalsOnly, 강도는 0~1. 시스템 설정에서 사용자가 글로벌 강도를 조정할 수 있다.

15 Assistive Access API

iOS 17에서 도입된 단순화 모드의 서드파티 통합. iOS 18에서 SwiftUI 기반의 AssistiveAccessSceneAdapter가 추가됐고, 2025년부터는 tvOS도 지원한다.

import SwiftUI

@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        // Assistive Access 모드에서 별도 단순화 UI 제공
        AssistiveAccessScene {
            SimpleHomeView()        // 큰 버튼 / 단일 작업 / 명확한 동사
        }
    }
}
설계 원칙 Assistive Access UI는 한 화면에 하나의 주된 작업, 큰 컨트롤(최소 화면 폭의 1/3), 단순한 동사 레이블, 되돌리기와 종료가 항상 보이도록 설계한다. 모달·딥 네비게이션·미세 제스처는 피한다.

16 Accessibility Nutrition Labels (App Store)

2025년 도입. App Store 앱 상세 페이지에 우리 앱이 어떤 접근성 기능을 지원하는지 사전에 표시되는 라벨.

App Store Connect에서 카테고리별로 자가 선언:

  • VoiceOver
  • Voice Control
  • Larger Text (Dynamic Type 11단계 전부 지원)
  • Sufficient Contrast (WCAG 2.x 수준)
  • Reduced Motion
  • Captions (영상에 자막 제공)
  • Differentiate Without Color
  • Switch Control / Full Keyboard Access
주의 "지원"으로 선언했는데 실제로 지원하지 않으면 거절·하향 노출 사유가 된다. 자가 선언 전에 XCTest 접근성 감사로 반드시 검증.

18 테스트 · Inspector · Audits

접근성은 "고쳤다고 믿었지만 안 고쳐진" 경우가 가장 흔한 분야다. 도구로 자동 검증하고, VoiceOver를 직접 켜고 써 본다.

Accessibility Inspector (macOS)

Xcode → Open Developer Tool → Accessibility Inspector. 시뮬레이터·실기기·Mac 앱 모두 검사 가능. Audit 탭에서 contrast, hit region, label, dynamic type, traits 누락 등을 자동 점검.

Xcode Accessibility Audit (XCTest)

iOS 17 / macOS 14부터 XCTest에 자동 감사가 들어왔다. UI 테스트 한 줄로 라벨 누락·동적 글자 잘림·히트 타깃 등을 검사.

func testHomeScreenAccessibility() throws {
    let app = XCUIApplication()
    app.launch()
    try app.performAccessibilityAudit()           // 모든 카테고리
    try app.performAccessibilityAudit(for: [.contrast, .dynamicType, .hitRegion])
}

VoiceOver 시뮬레이션 단축키

  • 실기기 VoiceOver 토글: Settings → Accessibility → Accessibility Shortcut에 VoiceOver 지정 후 측면 버튼 3회
  • 시뮬레이터: Features → Toggle Software Keyboard, Accessibility Inspector로 대체 권장
  • macOS VoiceOver: ⌘ + F5
  • VoiceOver 키: VO = Control + Option

접근성 디버깅용 환경 토글

Xcode → Environment Overrides에서 Bold Text·Increase Contrast·Reduce Motion·Reduce Transparency·Differentiate Without Color·Smart Invert·Larger Text를 라이브로 켜고 끌 수 있다.

Vision Pro 접근성

visionOS는 응시 + 핀치가 기본 입력이므로, 모든 인터랙티브 요소에 시각적 hover state와 충분한 hit target(권장 60pt 이상)이 필수다. VoiceOver는 카메라 기반 공간 디스크립션과 Live Recognition을 함께 제공한다.
