00 다섯 가지 관점

Apple의 접근성을 다섯 가지 관점에서 풀어 정리한 노트입니다. 어떤 관점으로 들어갈지 골라 들어가세요.

Part I
사용자 기능

VoiceOver · Zoom · Switch Control 등 사용자가 켜고 끄는 시스템 기능들. 각 기능이 왜 만들어졌고 무엇을 해결하는지부터.
Part II
개발자 API

UIKit · SwiftUI · AppKit의 접근성 API 카탈로그. 속성·모디파이어·환경값·테스트 도구.
Part III
기능 × SwiftUI

"Vision 카테고리에는 이런 SwiftUI API들이 있다" — 카테고리별 매핑. 그리고 API → 기능 역인덱스.
Part IV
피쳐별 레시피

"VoiceOver를 지원하려면 정확히 무엇을 해야 하는가" — 피쳐 27개의 최소 구현 · 확장 · 함정 · 검증.
Part V
부록

실무에서 가장 자주 빠뜨리는 12가지 체크리스트와 공식 레퍼런스 모음.

00 개요

Apple의 접근성은 4개 축 — Vision · Hearing · Mobility · Speech — 에 Cognitive가 더해진 5개 카테고리로 구분된다. 이 5개 카테고리는 시스템 설정의 구조이자, App Store Accessibility Nutrition Labels의 분류이기도 하다.

Vision
시각

VoiceOver, Zoom, Magnifier, Display & Text Size, Hover Typing, Spoken Content, Audio Descriptions.

Hearing
청각

Live Captions, Live Listen, MFi Hearing Devices, Sound Recognition, Music Haptics, RTT, Mono Audio.

Mobility
신체

Switch Control, Voice Control, AssistiveTouch, Eye Tracking, Head Tracking, Vocal Shortcuts, Full Keyboard Access.

Speech
발화

Personal Voice, Live Speech, Listen for Atypical Speech, Hold for Siri.

Cognitive
인지

Assistive Access, Guided Access, Reduce Motion, Background Sounds, Accessibility Reader.

2025 New
올해 신규

Accessibility Nutrition Labels, Magnifier for Mac, Braille Access, BCI for Switch Control, Name Recognition, Accessibility Reader, Enhanced View (visionOS).

읽는 법 Part I은 사용자 관점의 기능 목록입니다. Part II는 그 기능들과 상호작용하기 위한 개발자 API와 코드 예시입니다. 두 파트는 같은 기능을 양쪽에서 본 것이라고 생각하시면 됩니다.
